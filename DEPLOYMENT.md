# 设备管理系统 - 部署说明

## 🚀 快速部署

### 方法一：用户访问部署（推荐）
1. 双击运行 `deploy-user-access.bat`
2. 等待自动安装依赖和构建
3. 服务器启动后，用户可通过以下地址访问：
   - `http://192.168.0.136:5000` - 完整系统（前端+后端）- HTTP版本
   - `https://192.168.0.136:5001` - 完整系统（前端+后端）- HTTPS版本
   - `http://192.168.10.73:5000` - 完整系统（前端+后端）- HTTP版本
   - `https://192.168.10.73:5001` - 完整系统（前端+后端）- HTTPS版本

### 方法二：生产模式部署
1. 双击运行 `start-production.bat`
2. 等待自动安装依赖和构建
3. 服务器启动后，会显示所有可用的IP地址
4. 其他设备可通过任意一个IP地址访问（支持HTTP和HTTPS）

### 方法二：手动部署
```bash
# 1. 安装依赖
npm install
cd client && npm install && cd ..

# 2. 构建前端
cd client && npm run build && cd ..

# 3. 启动生产服务器
set NODE_ENV=production
npm start
```

## 🌐 局域网访问

服务器启动后会显示：
```

📊 本地访问地址: http://localhost:5000
🌐 局域网访问地址: http://192.168.1.100:5000
```

### 其他设备访问
1. **确保设备在同一局域网**
2. **在浏览器中访问**: `http://[服务器IP]:5000`
3. **例如**: `http://192.168.1.100:5000`

## 🔧 配置说明

### 端口配置
- 默认端口：5000
- 修改端口：设置环境变量 `PORT=8080`

### 数据库配置
- 确保MySQL服务正在运行
- 数据库配置在 `server/database.js` 中
- 默认配置：用户名=els, 密码=111111, 数据库=device_management

### 防火墙设置
如果其他设备无法访问，请检查：

#### Windows防火墙
1. 打开"Windows Defender 防火墙"
2. 点击"允许应用或功能通过Windows Defender 防火墙"
3. 点击"更改设置" → "允许其他应用"
4. 添加Node.js或允许端口5000

#### 命令行方式
```cmd
# 允许端口5000通过防火墙
netsh advfirewall firewall add rule name="Device Management System" dir=in action=allow protocol=TCP localport=5000
```

## 📱 移动设备访问

### 手机/平板访问
1. 确保设备连接到同一WiFi网络
2. 在浏览器中输入：`http://[服务器IP]:5000`
3. 系统会自动适配移动端界面

### 二维码访问（可选）
可以使用二维码生成工具生成访问链接的二维码，方便移动设备扫描访问。

## 🔒 HTTPS访问说明

### 协议支持
系统现在同时支持HTTP和HTTPS两种访问方式：
- **HTTP访问**：端口5000，适合内网环境，无需证书验证
- **HTTPS访问**：端口5001，提供加密传输，更安全

### HTTPS首次访问
由于使用自签名证书，首次访问HTTPS版本时浏览器会显示安全警告：

1. **Chrome/Edge**：
   - 点击"高级"或"Advanced"
   - 选择"继续访问"或"Proceed to site"

2. **Firefox**：
   - 点击"高级"
   - 选择"接受风险并继续"

3. **Safari**：
   - 点击"显示详细信息"
   - 选择"访问此网站"

### 测试HTTPS功能
访问测试页面：`http://192.168.0.136:5000/test-https` 或 `https://192.168.0.136:5001/test-https`

## 🔍 故障排除

### 常见问题

#### 1. 无法访问服务器
- 检查服务器是否正在运行
- 确认IP地址和端口是否正确
- 检查防火墙设置

#### 2. 数据库连接失败
- 确认MySQL服务正在运行
- 检查数据库配置信息
- 确认数据库用户权限

#### 3. 前端页面空白
- 确认前端已正确构建
- 检查 `client/build` 目录是否存在
- 查看浏览器控制台错误信息

### 日志查看
服务器运行时会显示详细日志，包括：
- 启动状态
- 数据库连接状态
- 访问请求日志
- 错误信息

## 🛡️ 安全建议

### 生产环境安全
1. **修改默认密码**：更改数据库密码
2. **限制访问**：配置防火墙规则
3. **HTTPS**：考虑使用SSL证书
4. **定期备份**：备份数据库数据

### 网络安全
- 仅在受信任的局域网内使用
- 避免在公网环境直接暴露
- 定期更新依赖包

## 📊 性能优化

### 服务器优化
- 使用PM2进行进程管理
- 配置Nginx反向代理
- 启用Gzip压缩

### 数据库优化
- 定期清理无用数据
- 创建适当的索引
- 监控数据库性能

## 🔄 更新部署

### 更新代码
1. 停止服务器（Ctrl+C）
2. 拉取最新代码
3. 重新运行 `start-production.bat`

### 数据备份
```bash
# 备份数据库
mysqldump -u els -p111111 device_management > backup.sql

# 恢复数据库
mysql -u els -p111111 device_management < backup.sql
```
