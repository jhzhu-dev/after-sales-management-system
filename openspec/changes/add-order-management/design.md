# Design: 订单管理核心 (Order Management Core)

## 1. 数据库概要设计
建立了 7 张关联表以支持订单的全生命周期：
- `orders`: 主表，记录基本成交信息、金额及当前进度。
- `order_payments`: 多期付款记录。
- `order_devices`: 订单按台设备清单（每台一条，含 SN）。
- `order_hardware_configs` / `order_software_configs`: 每个产品的精细化定制配置。
- `order_shipping_info`: 出货物流信息。
- `order_progress`: 生产阶段流转记录（按设备，含 SOP 检查 JSON）。

## 2. 后端逻辑
- **业务流程控制**: 采用 SOP 驱动的阶段流转（装配 -> 部署 -> 调试 -> 打包 -> 物流 -> 完成）。
- **审核机制**: 每个阶段开始需审核员批准，确保质量受控。
- **设备级追踪**: 生产进度与阶段推进按设备记录，支持按台交付。
- **事务处理**: 创建订单时采用原子性事务，确保主表与配置表同步成功。

## 3. 前端实现核心
- **多标签详情页**: 在单个页面内集成财务、配置、进度、物流等多个功能维度。
- **动态产品选择器**: 实现了“产品线 -> 产品 -> 数量”的树形动态增项逻辑，支持不限次数的产品添加。
- **订单概述同步**: 订单概述与数量根据设备清单自动汇总，编辑模式一致化展示。
- **生产中心联动**: 时间线与“完成阶段”按钮按设备当前进度工作。
- **自动日期管理**: 通过前端逻辑实现创建日期自动获取与同步。
- **API 通讯优化**: `baseURL` 统一采用相对路径 `/api` 以完美支持后端代理与 CORS 安全策略。
