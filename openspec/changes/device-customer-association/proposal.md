# Proposal: 设备-客户-订单三方关联 (Device-Customer-Order Association)

## 1. 背景
在原有的系统中，设备（Device）、客户（Customer）和订单（Order）是相对独立的实体。
- 设备记录设备自身的基本信息和子模块状态。
- 客户记录客户的基础联络信息。
- 订单记录销售合意及生产进度，虽然涉及产品，但未直接与具体的物理设备挂钩。

**痛点**: 无法快速追踪一台特定编号的物理设备归属于哪个客户，由哪个订单产生，以及该订单在生产时的具体硬件/软件配置。

## 2. 目标
通过建立三方关联，实现：
1. **资产归属追溯**: 每台设备都能明确其所属客户，方便售后跟进。
2. **生产质量追溯**: 建立了设备到订单的关联，从而能反向查询到该设备生产时的所有合意要求和配置记录。
3. **管理闭环**: 从销售下单（Order）到资产交付（Customer）再到全生命周期维护（Device）的闭环管理。

## 3. 核心方案
- 在数据库 `devices` 表中引入 `customer_id` 和 `order_id`。
- 后端 API 支持关联查询和多维度过滤。
- 前端实现 "绑定物理设备" 功能，将生产出的设备序列号填入订单。
- 并在客户视图中集中展示其名下的设备清单。
